# 名词解释

<Content />

本文介绍 TapData 常见的术语，帮助您快速理解产品和功能概念。

## 全量数据同步

即数据库迁移/数据库克隆，在数据流任务中，各种库级别数据源之间的数据全量迁移，适用于实例数据迁移、数据上下云迁移、数据库拆分扩容等业务场景。

## 增量数据同步

在数据流任务中，多个数据源之间通过特定的关联关系或处理的数据实时同步，适用于在不影响用户业务的情况下，满足用户对数据的分析、处理、灾备等方面的场景。

## 数据源

从外部连接到TapData系统的数据来源。当前支持的数据源主要指数据库（Database），后续将逐步增加File、GridFS、RestAPI、Dummy、Custom、UDP、Cache等类型。

## 数据复制

又指数据库复制/数据库克隆，在数据流任务中，各种库级别数据源之间的数据全量或实时增量迁移，适用于实例数据迁移、数据上下云迁移、数据库拆分扩容等业务场景。



## 数据转换

在数据流任务中，多个表或其他类型数据类型之间通过特定的关联关系或处理的数据实时同步，适用于在不影响用户业务的情况下，满足用户对数据的分析、处理、灾备等方面的场景



## 数据服务

在数据流任务中，将一个或多个表不同的字段生成新的模型以API的方式发布到外部环境，用户可通过API主动的获取数据内容。



## 连接

又称数据源，即从外部连接到TapData系统的数据库。当前支持的连接包含：mysql、Oracle、MongoDB、 SQL Server、PostgreSQL、Kafka、 Redis 等。



## 节点

指的是数据任务的编排页面里所拖选的数据源、处理方式的统称。



## 处理节点

指的是为满足数据同步需求的各种处理功能的节点，当前支持的处理节点包含：JavaScript/Java处理、数据库表过滤、字段处理、行级处理等。



## 源节点

指的是数据任务中，任意两个相邻连接的节点中，位置处于源头/生出连线的节点。



## 目标节点

指的是数据任务中，任意两个相邻连接的节点中，位置处于目标/被连接指向的节点。



## 共享挖掘

指增量日志的共享，当开启功能后，共享挖掘会挖掘增量日志，当多个增量任务都从同一个源获取增量日志时不需要重复开启日志采集进程，能极大缓解源库资源的占用和浪费



## 共享缓存

指将表中部分常用数据存放到缓存中供不同任务进行调用及处理，不用再从源端获取数据从而提高效率。


## 初始化

数据迁移或同步的任务中，对数据源节点已存在的数据进行迁移或同步的模式。

## TapData Agent

是指运行同步任务的执行程序，负责从管理端获取任务、连接源数据源、执行数据转换、输出到目标数据源。

## TCM 管理端

TapData Cloud 理控制台，让用户可以自主定义编排同步任务，并下发同步任务给到同步实例。

## TapData 引擎

TapData 的核心执行引擎，作为独立进程负责执行数据同步任务（包括全量同步和增量同步）以及数据验证任务（包括全量验证和增量验证），确保数据流转的准确性和实时性。

## TapData 管理器

TapData 的管理控制服务，作为独立进程为用户提供图形化界面，支持任务定义和管理、API 配置和启用等交互式操作，是用户管理和使用 TapData 的主要入口。


## Schema

源和目标数据连接模型。

## Database name

源和目标数据库的连接名称。


## QPS

每秒查询数（Queries Per Second），指数据同步任务每秒处理的输入输出变更事件的平均数量，用于衡量数据同步处理的性能指标。

## 增量验证

在增量同步过程中，TapData 对目标数据库中的记录与源数据库中的记录进行抽样比较的数据一致性验证功能。启用后，该功能会在增量同步期间持续运行，确保数据的准确性。更多介绍，见[增量数据校验](../user-guide/incremental-check.md)。


## API 服务器

TapData 提供的数据服务发布平台，支持将表数据以 RESTful API 形式对外发布。企业可通过 API 服务器构建统一的数据服务平台，为各类应用（如推送服务、移动应用、第三方系统等）提供标准化的数据访问接口，实现数据的安全共享和高效利用。

## 基础数据模型（FDM）

FDM（Foundation Data Model）是 TapData 的平台缓存层，可将源库的表数据提前同步至统一的平台缓存层，避免在后续执行复杂数据加工时直接读取/操作源库的数据。FDM 支持调整表字段、计算字段、合并表等操作，配合共享挖掘功能，可极大降低对源库所属业务的影响，确保源库性能稳定。更多介绍，见[数据服务平台模式介绍](../user-guide/real-time-data-hub/daas-mode/enable-daas-mode.md)。

## 主数据模型（MDM）

MDM（Master Data Model）是 TapData 的主数据管理层，通过数据分类帮助团队组织和标准化业务实体。用户可基于 MDM 定义清晰的数据结构，包括用户画像、订单详情、支付交易、风险评分模型等关键业务数据，形成统一的企业级数据视图，为业务应用提供标准化的数据服务。更多介绍，见[数据服务平台模式介绍](../user-guide/real-time-data-hub/daas-mode/enable-daas-mode.md)。
